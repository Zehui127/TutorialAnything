Hash Table

When to use: Look for problems that involve lookups, checking for duplicates, or counting frequencies. If you need to quickly check for the existence of a value, a hash table (or hash set) is often the most efficient tool.

Strategy: The core idea is to trade space for time. By using extra memory to store data in a hash table, you can achieve O(1) average time complexity for insertions, deletions, and lookups. For "Two Sum," we store numbers and their indices to find the complement in constant time. 

# General Structure for Hash Table lookups
hash_map = {}
for item in data:
    # Check if a complement/related item exists
    if (target - item) in hash_map:
        # Found a solution, do something
        return [hash_map[target - item], item]
    # Add current item to map for future lookups
    hash_map[item] = index

Two Pointers

When to use: This pattern is ideal for problems involving sorted arrays or linked lists where you need to find a pair of elements that meet certain criteria. It's also great for finding palindromes or reversing sequences.

Strategy: Use two pointers that move through the data structure. They can start from opposite ends and move towards each other (like in "Container With Most Water"), or one can be a "fast" pointer and the other "slow" (like in "Linked List Cycle"). This avoids nested loops and often leads to an O(n) solution.

# General Structure for Two Pointers (from ends)
left, right = 0, len(array) - 1
while left < right:
    # Do some logic with array[left] and array[right]
    if condition_met(array[left], array[right]):
        # Move one of the pointers
        left += 1
    else:
        # Move the other pointer
        right -= 1

Sliding Window

When to use: For problems that ask for the longest/shortest/best subarray or substring that satisfies a condition. Think "Longest Substring Without Repeating Characters" or "Minimum Window Substring."

Strategy: Define a "window" with two pointers, `left` and `right`. Expand the window by moving `right`. When the window no longer meets the criteria, shrink it from the left by moving `left`. This is much more efficient than generating all possible subarrays, usually resulting in O(n) time complexity.

# General Structure for Sliding Window
left = 0
result = 0
for right in range(len(array)):
    # Add array[right] to the window state

    # While the window is invalid...
    while window_is_invalid:
        # Remove array[left] from the window state
        left += 1

    # Update result with the valid window
    result = max(result, right - left + 1)

Linked List

When to use: Problems that explicitly involve linked list manipulation, such as reversal, merging, cycle detection, or removing nodes.

Strategy: Keep track of `head`, `current`, `previous`, and `next` nodes carefully. A dummy head node can simplify edge cases for insertions or deletions at the beginning of the list. For cycle detection, the fast/slow pointer approach is classic. For reversals, you'll need to juggle pointers to reverse the links iteratively.

# General Structure for Linked List traversal
current = head
while current:
    # Do something with current.val
    current = current.next

Tree / DFS / BFS

When to use: Any problem involving a tree or graph structure. Common tasks include traversal, finding paths, calculating depth/height, or level-order operations.

Strategy:

  • DFS (Depth-First Search): Goes deep down one path before exploring others. It's often implemented with recursion or an explicit stack. Great for pathfinding, checking connectivity, and problems where you need to explore all possibilities in a branch.
  • BFS (Breadth-First Search): Explores level by level. It's implemented with a queue. Perfect for finding the shortest path in an unweighted graph/tree and for level-order traversals.

# General Structure for DFS (Recursive)
def dfs(node):
    if not node:
        return
    # Pre-order logic here
    dfs(node.left)
    # In-order logic here
    dfs(node.right)
    # Post-order logic here

# General Structure for BFS
from collections import deque
def bfs(root):
    queue = deque([root])
    while queue:
        node = queue.popleft()
        # Process node
        if node.left: queue.append(node.left)
        if node.right: queue.append(node.right)

Dynamic Programming (DP)

When to use: Look for problems asking for the optimal (max/min/longest/shortest) way to do something, where the problem can be broken down into overlapping subproblems. "Climbing Stairs," "Coin Change," and "Longest Increasing Subsequence" are classic DP examples.

Strategy:

  1. Identify Subproblems: Define `dp[i]` or `dp[i][j]` to represent the solution for a smaller version of the problem.
  2. Find the Recurrence Relation: Figure out how to solve `dp[i]` using the solutions to previous subproblems (e.g., `dp[i-1]`, `dp[i-2]`).
  3. Identify Base Cases: Determine the solution for the smallest subproblems (e.g., `dp[0]`, `dp[1]`).
You can solve it using memoization (top-down recursion with caching) or tabulation (bottom-up iteration). 

# General Structure for DP (Tabulation, 1D)
dp = [0] * (n + 1)
# Base cases
dp[0] = 1
# Fill the DP table using the recurrence relation
for i in range(1, n + 1):
    dp[i] = dp[i-1] + dp[i-2] # e.g., Fibonacci
return dp[n]

Backtracking

When to use: Problems that ask for all possible solutions or combinations, like "Generate Parentheses," "Permutations," and "Subsets."

Strategy: This is a form of recursion. Build a solution step by step. At each step, explore all valid choices. If a choice leads to a dead end, "backtrack" by undoing the choice and trying another. A helper function with the current state (e.g., the current permutation being built) is a common pattern.

# General Structure for Backtracking
def backtrack(current_state, ...):
    if solution_found(current_state):
        add_solution_to_results()
        return

    for choice in possible_choices:
        # 1. Choose: Add choice to state
        current_state.add(choice)
        # 2. Explore: Recurse with new state
        backtrack(current_state, ...)
        # 3. Unchoose: Remove choice to backtrack
        current_state.remove(choice)

1. Hash Table: Two Sum (#1)

Problem: Given an array of integers `nums` and a `target`, return indices of the two numbers that add up to `target`.

Strategy: This is a classic Hash Table problem. We iterate through the array once, storing each number and its index in a hash map. For each number, we calculate its complement (`target - num`) and check if the complement already exists in the map. If it does, we've found our pair.

class Solution:
    def twoSum(self, nums: list[int], target: int) -> list[int]:
        num_map = {}
        for i, num in enumerate(nums):
            complement = target - num
            if complement in num_map:
                return [num_map[complement], i]
            num_map[num] = i

2. Sliding Window: Longest Substring Without Repeating Characters (#3)

Problem: Given a string `s`, find the length of the longest substring without repeating characters.

Strategy: This is a perfect problem for the Sliding Window technique. We use two pointers, `left` and `right`, to define our window, and a set to track characters in the window. We expand the window by moving `right`. If we find a character that's already in the set, we shrink the window from the left until the duplicate is gone. We continuously update our max length.

class Solution:
    def lengthOfLongestSubstring(self, s: str) -> int:
        char_set = set()
        left = 0
        max_length = 0
        for right in range(len(s)):
            while s[right] in char_set:
                char_set.remove(s[left])
                left += 1
            char_set.add(s[right])
            max_length = max(max_length, right - left + 1)
        return max_length

3. Two Pointers: Container With Most Water (#11)

Problem: Given an array of non-negative integers representing heights, find two lines that together with the x-axis form a container, such that the container contains the most water.

Strategy: Use the Two Pointers pattern. Start with one pointer at the beginning (`left`) and one at the end (`right`). Calculate the area. The area is limited by the shorter of the two lines. To potentially find a larger area, you must move the pointer of the shorter line inward, because moving the taller line's pointer can't increase the height and will only decrease the width.

class Solution:
    def maxArea(self, height: list[int]) -> int:
        left, right = 0, len(height) - 1
        max_area = 0
        while left < right:
            h = min(height[left], height[right])
            w = right - left
            max_area = max(max_area, h * w)
            if height[left] < height[right]:
                left += 1
            else:
                right -= 1
        return max_area

4. Linked List: Reverse Linked List (#206)

Problem: Given the `head` of a singly linked list, reverse the list, and return the reversed list's head.

Strategy: This is a fundamental Linked List manipulation. You iterate through the list, and for each node, you reverse its `next` pointer to point to the `previous` node. You'll need three pointers: `prev` (starts as `None`), `curr` (starts as `head`), and a `next_temp` to hold the next node before you overwrite the pointer.

# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def reverseList(self, head: ListNode) -> ListNode:
        prev = None
        curr = head
        while curr:
            next_temp = curr.next
            curr.next = prev
            prev = curr
            curr = next_temp
        return prev

5. Dynamic Programming: Climbing Stairs (#70)

Problem: You are climbing a staircase. It takes `n` steps to reach the top. Each time you can either climb 1 or 2 steps. In how many distinct ways can you climb to the top?

Strategy: This is a classic Dynamic Programming problem. The number of ways to get to step `n` is the sum of the ways to get to step `n-1` (and taking one step) and the ways to get to step `n-2` (and taking two steps). This is the Fibonacci sequence. We can build a DP array where `dp[i]` stores the number of ways to reach step `i`.

class Solution:
    def climbStairs(self, n: int) -> int:
        if n <= 2:
            return n
        # We only need to store the last two values
        one_step_before = 2
        two_steps_before = 1
        for i in range(3, n + 1):
            current = one_step_before + two_steps_before
            two_steps_before = one_step_before
            one_step_before = current
        return one_step_before

6. Backtracking: Generate Parentheses (#22)

Problem: Given `n` pairs of parentheses, write a function to generate all combinations of well-formed parentheses.

Strategy: Use Backtracking. We build the string from left to right. We can add an opening parenthesis if we haven't used all `n` of them. We can add a closing parenthesis if it wouldn't exceed the number of open parentheses already placed. We recurse until the string length is `2*n`.

class Solution:
    def generateParenthesis(self, n: int) -> list[str]:
        result = []
        def backtrack(s = '', left = 0, right = 0):
            if len(s) == 2 * n:
                result.append(s)
                return
            if left < n:
                backtrack(s + '(', left + 1, right)
            if right < left:
                backtrack(s + ')', left, right + 1)
        backtrack()
        return result

7. Tree / DFS: Maximum Depth of Binary Tree (#104)

Problem: Given the root of a binary tree, return its maximum depth.

Strategy: A perfect use case for DFS (Depth-First Search) via recursion. The depth of a tree is `1 + max(depth of left subtree, depth of right subtree)`. The base case for the recursion is when a node is `None`, in which case its depth is 0.

# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def maxDepth(self, root: TreeNode) -> int:
        if not root:
            return 0
        left_depth = self.maxDepth(root.left)
        right_depth = self.maxDepth(root.right)
        return 1 + max(left_depth, right_depth)

8. Binary Search: Search in Rotated Sorted Array (#33)

Problem: You are given an integer array `nums` sorted in ascending order, and it has been rotated at some unknown pivot. Find the index of `target` if it is in `nums`, or -1 otherwise.

Strategy: This requires a modified Binary Search. At each step, find the middle element. Check which half of the array (from `left` to `mid` or `mid` to `right`) is sorted. Then, check if the `target` lies within the range of that sorted half. If it does, search that half; otherwise, search the other half.

class Solution:
    def search(self, nums: list[int], target: int) -> int:
        left, right = 0, len(nums) - 1
        while left <= right:
            mid = (left + right) // 2
            if nums[mid] == target:
                return mid
            # Check if left half is sorted
            if nums[left] <= nums[mid]:
                if nums[left] <= target < nums[mid]:
                    right = mid - 1
                else:
                    left = mid + 1
            # Otherwise, right half must be sorted
            else:
                if nums[mid] < target <= nums[right]:
                    left = mid + 1
                else:
                    right = mid - 1
        return -1

9. Heap: Kth Largest Element in an Array (#215)

Problem: Given an integer array `nums` and an integer `k`, return the `k`th largest element in the array.

Strategy: A Heap (or Priority Queue) is perfect for this. We can use a min-heap of size `k`. Iterate through the numbers. If the heap has fewer than `k` elements, add the number. If the heap is full and the current number is larger than the smallest element in the heap (the root), pop the root and push the current number. After iterating, the root of the heap is the `k`th largest element.

import heapq
class Solution:
    def findKthLargest(self, nums: list[int], k: int) -> int:
        min_heap = []
        for num in nums:
            if len(min_heap) < k:
                heapq.heappush(min_heap, num)
            elif num > min_heap[0]:
                heapq.heapreplace(min_heap, num)
        return min_heap[0]

10. Stack: Valid Parentheses (#20)

Problem: Given a string `s` containing just the characters '(', ')', '{', '}', '[' and ']', determine if the input string is valid.

Strategy: Use a Stack. Iterate through the string. If you see an opening bracket, push it onto the stack. If you see a closing bracket, check if the stack is empty or if the top of the stack is the corresponding opening bracket. If it is, pop the stack. If not, the string is invalid. After the loop, the string is valid only if the stack is empty.

class Solution:
    def isValid(self, s: str) -> bool:
        stack = []
        mapping = {")": "(", "}": "{", "]": "["}
        for char in s:
            if char in mapping:
                top_element = stack.pop() if stack else '#'
                if mapping[char] != top_element:
                    return False
            else:
                stack.append(char)
        return not stack
